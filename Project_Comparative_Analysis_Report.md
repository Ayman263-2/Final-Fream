### أولاً: مقارنة البنية المعمارية العامة (System Architecture Comparison)

يوضح هذا القسم كيف تحول المشروع من مسودة برمجية خطية واحدة في دفاتر Colab إلى بنية خدمات برمجية مخصصة وموزعة (Microservices Architecture) لتحقيق الإنتاجية القصوى.

```
+-------------------------------------------------------------------------------------------------+
|                               المقارنة المعمارية الهيكلية                                      |
+-----------------------------------+-------------------------------------------------------------+
| النسخ التجريبية القديمة (Colab Drafts) | الكود النهائي المستقر المعتمد (Final Production Code)       |
+-----------------------------------+-------------------------------------------------------------+
| 1. معمارية أحادية كتلتية (Monolithic)| 1. معمارية خدماتية مفصولة المسؤولية (Decoupled Microservice)  |
|    برنامج واحد طويل يحتوي على الكشف،|    تقسيم النظام لملفات مخصصة:                              |
|    والتتبع، واللوجيك، وحفظ الفيديو   |    - server.py: خادم FastAPI لإدارة الطلبات والـ API.      |
|    بشكل متسلسل وتتابعي خطي.       |    - logic.py: المحرك الجوهري لمعالجة الفيديو والفيزياء.  |
|                                   |    - config.py: المعايير المفرزة ومعاملات الضبط للسهولة.    |
|                                   |    - utils.py: التحقق من الاعتماديات ونفق البث السحابي.   |
+-----------------------------------+-------------------------------------------------------------+
| 2. تشغيل أحادي الخيط (Single-Thread)| 2. معمارية متوازية متعددة الخيوط (Multithreaded Pipeline)   |
|    يتوقف البرنامج عن القراءة عند  |    فصل استخراج الإطارات (Reader Thread) عن التحليل المنطقي   |
|    حساب مصفوفة الاستدلال، مما يسبب |    (Writer Thread) باستخدام طابور FIFO آمن متزامن لضمان  |
|    تقطعاً (Lag) وسقوطاً في الإطارات. |    استقرار الـ FPS الفعلي في الوقت الفعلي.                  |
+-----------------------------------+-------------------------------------------------------------+
| 3. معاملات ضبط صلبة (Hardcoded)    | 3. مركزية التكوين والإعداد الديناميكي (Dynamic Config)     |
|    كتابة عتبات الثقة، والمنافذ،   |    وضع جميع المعاملات الفيزيائية والكشفية داخل config.py    |
|    وأسماء النماذج في قلب الدوال.  |    لتعديل النظام فورياً دون الحاجة للمساس باللوجيك المعقد.  |
+-----------------------------------+-------------------------------------------------------------+
```

### ثانياً: تتبع المركبات وتصفية المسارات (Vehicle Tracking & Trajectory Smoothing)

مقارنة بين دقة تتبع مسار السيارات والتقنيات الرياضية المستخدمة لتصفية الضوضاء البصرية الناتجة عن الكاشف:

```
+-------------------------------------------------------------------------------------------------+
|                               مقارنة تتبع مسار المركبات                                        |
+-----------------------------------+-------------------------------------------------------------+
| النسخ التجريبية القديمة (Colab Drafts) | الكود النهائي المستقر المعتمد (Final Production Code)       |
+-----------------------------------+-------------------------------------------------------------+
| 1. تتبع افتراضي عام (Native Tracker) | 1. تتبع BoT-SORT مخصص وثابت (BoT-SORT + Re-ID)              |
|    استدعاء خيار التتبع المدمج في  |    تحميل ملف تكوين botsort.yaml مخصص يدمج ميزات الحركة       |
|    نموذج YOLO دون أي تخصيص      |    (Motion) ومعلومات المظهر (Appearance) لربط المعرفات      |
|    للمعاملات الحساسة للشارع.      |    حتى عند التقارب الشديد للمركبات.                         |
+-----------------------------------+-------------------------------------------------------------+
| 2. حساب إزاحة خطي بسيط ($dx, dy$)   | 2. تصفية المسار التاريخي بمرشح كالمان (Kalman Filter Tracker) |
|    حساب موضع مركز السيارة بين      |    تخصيص فلتر كالمان رياضي لكل مركبة لتقدير حالتها الفيزيائية |
|    إطارين متتاليين بشكل عشوائي، مما|    (الموضع والسرعة) وتنعيم المسار وحساب التسارع الدقيق:    |
|    أدى لقفزات سرعة خاطئة ومفاجئة.  |    $$x_t = F x_{t-1} + w_t$$                                |
+-----------------------------------+-------------------------------------------------------------+
| 3. تنعيم أسي بسيط (EMA Only)      | 3. تنعيم بفلتر كالمان + تدفق بصري محلي للمركبة              |
|    استخدام معادلة ألفا فقط لتنعيم |    دمج إزاحة كالمان مع التدفق البصري المحلي (Optical Flow)  |
|    صورة المربعات مما قلل الرجة    |    للمركبة للتعويض الفوري وضمان ثبات مؤشر الحركة والتسارع   |
|    البصرية لكنه لم يمنع القفزات.   |    تحت الضوضاء العالية.                                     |
+-----------------------------------+-------------------------------------------------------------+
```

### ثالثاً: تعويض اهتزاز الكاميرات (Global Motion Compensation - GMC)

المقارنة الرياضية لمعالجة اهتزازات الكاميرا الخارجية المثبتة على أعمدة الإنارة بالطرق السريعة:

```
+-------------------------------------------------------------------------------------------------+
|                                 مقارنة مقاومة اهتزاز الكاميرا                                    |
+-----------------------------------+-------------------------------------------------------------+
| النسخ التجريبية القديمة (Colab Drafts) | الكود النهائي المستقر المعتمد (Final Production Code)       |
+-----------------------------------+-------------------------------------------------------------+
| 1. غياب كامل لمعاوضة الحركة       | 1. معالج معاوضة الحركة العام المدمج (`GlobalMotionComp`)   |
|    تأثر النظام بجميع اهتزازات     |    استخدام خوارزمية Lucas-Kanade للتدفق البصري على زوايا   |
|    الكاميرا الناتجة عن الرياح أو   |    الصورة الثابتة لتقدير مصفوفة التحويل الروتيني (Affine):  |
|    حركة الجسور والرافعات المجاورة. |    $$M = \begin{bmatrix} a & b & tx \\ c & d & ty \end{bmatrix}$$ |
+-----------------------------------+-------------------------------------------------------------+
| 2. إنذارات كاذبة مرتفعة للغاية    | 2. استقرار السرعات والصفر الفيزيائي للسيارات                |
|    تفسير حركة البكسلات الناتجة عن |    عندما تهتز الكاميرا، يتم تعويض الإزاحة وتطبيق المصفوفة  |
|    الاهتزاز كسرعة حقيقية للمركبات  |    العكسية على سرعة السيارات، لتظل سرعة السيارات الثابتة     |
|    المتوقفة، مما يطلق إنذارات كاذبة.|    صفراً حقيقياً تماماً دون أي انحراف.                     |
+-----------------------------------+-------------------------------------------------------------+
```

### رابعاً: منطق كشف التصادم ودمج القرارات (Decision Logic & Collision Detection)

```

|                              مقارنة منطق التحليل وكشف الحوادث                                  |

|    مؤقتات الوقوف، مما تسبب بانهيار|    الأزمان، مما يضمن ثبات الكشف سواء تم تشغيل الفيديو       |
|    المنطق عند تأخر معالجة الخادم. |    بسرعة 5 إطارات/ثانية أو 60 إطاراً/ثانية.                   |
```
